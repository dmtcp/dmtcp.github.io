DMTCP Plugin Interface:

DMTCP modules provide a flexible way to write an add-on library that can DMTCP event hooks, add custom wrappers around system calls, and add a custom distributed nameservice facility using the DMTCP Coordinator. DMTCP module examples can be found in test/plugin directory of the source distribution.

DMTCP Aware Programming Interface:

If you wish to call DMTCP from within your checkpointed program, we provide an optional programming interface called DMTCP Aware. If you are using binary distribution packages, you need to install libdmtcpaware1 (for dynamic linking) or libdmtcpaware-dev (for static linking) along with the dmtcp package. There are several ways to use DMTCP Aware:

  • Static linkage: Add dmtcpaware.h and libdmtcpaware.a to your project. If you are using libdmtcpaware-dev binary distribution, dmtcpaware.h can be found in the /usr/include/, and libdmtcpaware.a can be found in the /usr/lib/ directory. Look at the dmtcpaware directory of the source distribution for more details.
  • Dynamic linkage: Add dmtcpaware.h to your source code and link with -ldmtcpaware. The packages libdmtcpaware1 and libdmtcpaware-dev should be installed on your system. The libdmtcpaware.so.* files can also be found in the .../dmtcp/src/ directory of the source distribution.
  • Include dmtcpaware.h from your code. You should now have access to the DMTCP Aware API.
  • Documentation of the DMTCP Aware API functions is available. To test if your program is running under dmtcp_checkpoint use dmtcpIsEnabled(). Note that if your program is not running under DMTCP, dmtcpIsEnabled() will return false, and all other API functions will return -128 or NULL and have no effect.

Python Interface:

DMTCP also provides a simple Python module for accessing DMTCP functions within a Python session. You can import the Python module found as contrib/python/dmtcp.py. Other Python utilities are also in contrib/python/.
